This is a test for the open source rendering engine WebWhirr.

WebWhirr is, right now, in a very early stage of development. In a very crude way, it reads two different tags in HTML: the "p" and "b" tags.

Here are a lot of words that I'm typing to test the new wrapping feature I just put in. Below is WebWhirr's logo.

At this point, you may be wondering: How does a rendering engine work? For an incredibly basic overview, let's look at it this way: As the document is parsed, objects called "nodes" are created. These nodes contain all of the information necessary to display a certain part of a web page. These nodes are then taken and linked together as a linked list. This list is passed to the painter, which takes the information in the nodes and displays the web page.

This is another line that is here to test nested tags like the bold tag. It demonstrates what happens when there is more than one b tag within a single node.
